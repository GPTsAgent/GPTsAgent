# Support

Use GitHub Issues for:

- GPT Builder setup questions.
- Sandbox File Operator package defects.
- Validation failures.
- Documentation gaps.
- Public-safe feature proposals.

Use the public Telegram group for quick discussion, lightweight coordination, or early feedback:

- https://t.me/GPTsAgentChat

Use Telegram for sanitized conversation only. Move the result into GitHub Issues or Pull Requests when it needs tracking, review, or a permanent record.

If a Telegram conversation turns into a concrete contribution idea, open the Contribution Idea issue template with a sanitized summary before work starts.

For channel rules, label meanings, and Telegram-to-GitHub conversion, see `docs/COMMUNITY-PLAYBOOK.md`.

Do not use public issues for real secrets, private repository contents, customer data, production incidents, or credential bodies.

For the fastest triage, include:

- Package version.
- Affected file or workflow.
- What you expected.
- What happened.
- Which validation command ran.
- Whether any result is `NOT RUN`, `NOT VERIFIED`, or `TIMEOUT`.
