# Support

Use GitHub Issues for:

- GPT Builder setup questions.
- Sandbox File Operator package defects.
- Validation failures.
- Documentation gaps.
- Public-safe feature proposals.

Do not use public issues for real secrets, private repository contents, customer data, production incidents, or credential bodies.

For the fastest triage, include:

- Package version.
- Affected file or workflow.
- What you expected.
- What happened.
- Which validation command ran.
- Whether any result is `NOT RUN`, `NOT VERIFIED`, or `TIMEOUT`.
